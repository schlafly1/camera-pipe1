    The JPEG-per-source problem: multifilesink writes frames in the order they    
  arrive, interleaved across cameras. The source_id in the ObjectDetector probe 
  tells you which camera triggered, so the VLM worker needs to find a JPEG that 
  corresponds to that camera. The simplest fix is to write one multifilesink per
   source using a tee — but pyservicemaker's .link() doesn't expose tee
  natively.

  Practical recommendation for now: run N separate processes (one per camera).  
  The per-process overhead is small — TRT engine is cached on disk and
  deserialized fast, and the inference runs on GPU which is shared anyway. With 
  5–10 cameras on a Jetson Orin, this works fine. When you hit GPU memory
  limits, switch to the batched single-pipeline approach.
